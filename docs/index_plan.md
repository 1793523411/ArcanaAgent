Plan to implement                                                                                                                                                                                                              │
│                                                                                                                                                                                                                                │
│ Code Index Strategy System — 代码索引策略系统                                                                                                                                                                                  │
│                                                                                                                                                                                                                                │
│ Context                                                                                                                                                                                                                        │
│                                                                                                                                                                                                                                │
│ 当前 agent 分析大型代码库时，完全依赖运行时探索（ripgrep + 文件读取），无预建索引、无 AST 解析、无依赖图。调研（见 docs/research-ai-coding-tools-code-understanding.md）发现业界有三种成熟路线：                               │
│                                                                                                                                                                                                                                │
│ 1. Repo Map（Aider 方式）：Tree-sitter AST → 符号图 → PageRank 排序，效果最佳的单一改进                                                                                                                                        │
│ 2. 向量检索（Continue.dev / Cursor 方式）：嵌入 + 向量库，适合超大仓库语义搜索                                                                                                                                                 │
│ 3. 运行时探索优化：不建索引，但通过更智能的工具和提示提升效率                                                                                                                                                                  │
│                                                                                                                                                                                                                                │
│ 用户希望采用策略模式，三种方案都做，用户可选择，系统默认推荐最佳方案。                                                                                                                                                         │
│                                                                                                                                                                                                                                │

│                                                                                                                                                                                                                                │
│ getSnapshot() 实现：                                                                                                                                                                                                           │

│ - 对结果做简单评分（精确匹配 > 部分匹配 > 文件名匹配）                                                                                                                                                                         │

│ 改动 2：RepomapStrategy — Tree-sitter + PageRank                                                                                                                                                                               │
│                                                                                                                                                                                                                                │
│ 新文件: server/src/index-strategy/strategies/repomap.ts                                                                                                                                                                        │
│                                                                                                                                                                                                                                │
│ 新 npm 依赖（加到 server/package.json）：                                                                                                                                                                                      │
│ - tree-sitter + tree-sitter-typescript + tree-sitter-python + tree-sitter-javascript — AST 解析                                                                                                                                │
│ - pagerank.js — PageRank 算法（纯 JS，无 native 依赖）                                                                                                                                                                         │
│                                                                                                                                                                                                                                │
│ ▎ 注：tree-sitter 有 native binding，需要系统安装编译工具。checkDependencies() 会检测是否可用。                                                                                                                                │
│                                                                                                                                                                                                                                │
│ buildIndex() 流程：                                                                                                                                                                                                            │
│ 1. 遍历工作区文件（跳过 node_modules/.git 等噪声目录）                                                                                                                                                                         │
│ 2. 对每个支持的文件类型，用 Tree-sitter 解析 AST                                                                                                                                                                               │
│ 3. 提取符号定义：函数名、类名、接口、类型、常量（含文件路径 + 行号）                                                                                                                                                           │
│ 4. 提取符号引用：扫描所有文件中对已定义符号的引用                                                                                                                                                                              │
│ 5. 构建有向图：文件为节点，文件A引用了文件B的符号 → A→B 的边（权重=引用次数）                                                                                                                                                  │
│ 6. 运行 PageRank，为每个符号和文件计算重要度分数                                                                                                                                                                               │
│ 7. 将结果缓存到 {workspace}/.agents/index/repomap.json，含每个文件的 mtime                                                                                                                                                     │
│                                                                                                                                                                                                                                │
│ getSnapshot() 实现：                                                                                                                                                                                                           │
│ - 按 PageRank 排序选出 top-N 文件和符号                                                                                                                                                                                        │
│ - 输出精简的项目地图：文件路径 → [符号名(类型)]                                                                                                                                                                                │
│ - 用二分搜索拟合 maxTokens 预算（默认 2048 tokens）                                                                                                                                                                            │
│                                                                                                                                                                                                                                │
│ search() 实现：                                                                                                                                                                                                                │
│ - 先在符号表中精确匹配 → 再做模糊匹配                                                                                                                                                                                          │
│ - 结果按 PageRank 分数加权排序                                                                                                                                                                                                 │
│                                                                                                                                                                                                                                │
│ incrementalUpdate() 实现：                                                                                                                                                                                                     │
│ - 检查文件 mtime，仅重新解析变更的文件                                                                                                                                                                                         │
│ - 更新图中该文件相关的边，重新跑 PageRank                                                                                                                                                                                      │
│                                                                                                                                                                                                                                │
│ 缓存格式 ({workspace}/.agents/index/repomap.json)：                                                                                                                                                                            │
│ {                                                                                                                                                                                                                              │
│   "version": 1,                                                                                                                                                                                                                │
│   "createdAt": "...",                                                                                                                                                                                                          │
│   "files": {                                                                                                                                                                                                                   │
│     "src/index.ts": {                                                                                                                                                                                                          │
│       "mtime": 1711000000000,                                                                                                                                                                                                  │
│       "symbols": [                                                                                                                                                                                                             │
│         { "name": "main", "type": "function", "line": 10, "rank": 0.85 }                                                                                                                                                       │
│       ]                                                                                                                                                                                                                        │
│     }                                                                                                                                                                                                                          │
│   },                                                                                                                                                                                                                           │
│   "graph": { "edges": [...] }                                                                                                                                                                                                  │
│ }                                                                                                                                                                                                                              │
│                                                                                                                                                                                                                                │

│ 3. 存入 LanceDB 表，存储在 {workspace}/.agents/index/vectors/                                                                                                                                                                  │

│ - 对 query 生成嵌入向量                                                                                                                                                                                                        │
│ - LanceDB 最近邻搜索，返回 top-K chunks                                                                                                                                                                                        │
│ - 可选：hybrid search — 向量结果 + ripgrep 关键词结果合并去重                                                                                                                                                                  │
│                                                                                                                                                                                                                                │
│ getSnapshot() 实现：                                                                                                                                                                                                           │
│ - 复用 NoneStrategy 的文件树扫描                                                                                                                                                                                               │
│ - 额外标注向量索引覆盖的文件数量                                                                                                                                                                                               │
│                                                                                                                                                                                                                                │
│ incrementalUpdate()：                                                                                                                                                                                                          │
│ - 检查 mtime，仅重新分块+嵌入变更的文件                                                                                                                                                                                        │
│ - 删除旧 chunk → 插入新 chunk                                                                                                                                                                                                  │
